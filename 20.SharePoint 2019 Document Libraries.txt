 **SharePoint 2019 Document Libraries**, here is the step-by-step guide for creating and managing libraries with best practices, 


**Step 1: Navigate to Your Site Collection**
Open the site collection where you want to add the library (for example, the **Logistics and Supply Chain** department site).

**Step 2: Add the Document Library App**
Click on **Site contents** or the gear icon to **Add an app**. From the available apps, select **Document Library**.

**Step 3: Use a Short Name Without Spaces (Crucial Step)**
When first creating the library, give it a **very short name without spaces** (e.g., "Logistics"). **Best Practice:** Avoid spaces during creation because SharePoint replaces them with "**%20**" in the URL, making the web address long and messy.

**Step 4: Rename the Library for Display**
Once the library is created with a clean URL, go to **Library settings** and change the name to a more descriptive one (e.g., "Logistics and Supply"). This updates the display name for users but **keeps the short, clean URL** you created in the previous step.

**Step 5: Access Default Metadata (Attributes)**
Understand that a document library acts like a database table with default fields. To see all hidden default attributes—such as **ID, Created By, File Size, and Check-in Comments**—go to the library view and enable all columns,.

**Step 6: Understand the Unique ID**
Every document or folder added is assigned a **unique, autogenerated ID** (Primary Key). Even if you delete an item, the ID is not reused; the next item will receive the next number in the sequence. This is essential for developers and for organizing data.

**Step 7: Customize the Library View**
If the default view is cluttered, click **Edit current view**. Uncheck unnecessary columns and **change the position numbers** (e.g., move the ID column to position 1) to make the information easier for your team to read.

**Step 8: Create Custom Columns**
To add specific data fields not provided by default, go to **Library settings** and click **Create column**. For example, you can create a **"Document Type"** column using the **Choice** type to let users label files as Letters, Books, or Magazines.

**Step 9: Choose the Correct Data Type**
Select from various column types based on your needs: **Single line of text, Date and Time, Currency, Person or Group** (to look up staff from Active Directory), or **Lookup columns** (to pull data from other lists),.

**Step 10: Set Mandatory and Unique Constraints**
When creating a column, you can specify if the information is **mandatory** (Required). If set to "Yes," users cannot save a document without filling that field. You can also enforce **unique values** to prevent duplicate data entries.

**Step 11: Update Document Properties**
After adding custom columns, users can select a document, click **Edit properties**, and fill in the new fields (like selecting "Book" from your dropdown menu) to properly categorize their files.
