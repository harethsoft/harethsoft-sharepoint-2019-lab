**SharePoint 2019 SSL Binding and HTTPS Configuration**

 transitioning a SharePoint site from HTTP to HTTPS, here is the step-by-step guide for configuring SSL-based bindings

**Step 1: Identify the Existing HTTP Configuration**
Verify that your current SharePoint site is running on **HTTP via port 80** by checking the site settings in **Central Administration** and the bindings in **IIS Manager**.

**Step 2: Access the Certificate Authority (CA) Server**
Ensure you have a **Certificate Authority server** role already installed and active within your environment to issue the necessary web certificates.

**Step 3: Open MMC on the Local Web Server**
On the server hosting the web application, open the **Microsoft Management Console (MMC)** and add the **Certificates snap-in** for the local computer account.

**Step 4: Request a New Certificate**
Navigate to the "Personal" certificates folder and select the option to **"Request New Certificate."** Use the **Active Directory Enrollment Policy** available to your domain-joined server.

**Step 5: Select the Web Server Template**
From the list of available certificate types, choose the **Web Server** template to ensure the certificate is compatible with IIS hosting.

**Step 6: Configure Certificate Details**
Under the certificate properties, define the **Common Name (Subject Name)**. You may use the server name (e.g., sp22 app), a specific **DNS name**, or the server's **IP address**.

**Step 7: Enable Private Key Exportability**
In the **Private Key** settings, ensure you select the option to make the **private key exportable**, which provides more flexibility for certificate management.

**Step 8: Enroll and Verify in IIS**
Complete the enrollment process and then open **IIS Manager**. Check the **Server Certificates** section to confirm that the new certificate (e.g., sp22 SSL) has been successfully issued.

**Step 9: Apply the HTTPS Binding**
Select your SharePoint site in IIS (e.g., SharePoint 85), click on **"Bindings,"** and add a new **HTTPS binding**. Select the newly created certificate from the dropdown menu.

**Step 10: Restart and Validate HTTPS Access**
**Restart the website** in IIS. Browse to the URL using the HTTPS protocol to confirm the site is now securely configured and accessible via **SSL-based binding**.
